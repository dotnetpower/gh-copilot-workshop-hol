# 프롬프트 엔지니어링 이해

## 프롬프트 엔지니어링이란?

프롬프트 엔지니어링은 AI 모델로부터 원하는 결과를 얻기 위해 효과적인 입력을 작성하는 기술입니다.

## 왜 중요한가?

### 1. 정확성 향상
- 명확한 프롬프트 → 정확한 코드
- 모호한 요청 → 예상치 못한 결과

### 2. 생산성 증대
- 첫 시도에서 원하는 결과 획득
- 반복 수정 시간 감소

### 3. 코드 품질 개선
- 베스트 프랙티스 적용 요청 가능
- 특정 패턴이나 스타일 지정 가능

## 기본 원칙

### 명확성 (Clarity)
```javascript
// 나쁜 예: 모호한 주석
// 함수 만들기

// 좋은 예: 명확한 주석
// 두 숫자를 입력받아 합계를 반환하는 함수를 작성
function add(a, b) {
    return a + b;
}
```

### 구체성 (Specificity)
```python
# 나쁜 예: 일반적인 요청
# 데이터 처리

# 좋은 예: 구체적인 요청
# CSV 파일에서 데이터를 읽어 판다스 DataFrame으로 변환하고,
# 'age' 컬럼의 평균값을 계산하는 함수
```

### 컨텍스트 제공
```typescript
// 기존 코드와 함께 사용될 컨텍스트 제공
interface User {
    id: number;
    name: string;
    email: string;
}

// User 배열에서 특정 이메일을 가진 사용자를 찾는 함수
```

## 실습

다음 요구사항으로 프롬프트를 작성해보세요:

1. **명확하게**: "배열 정렬" → "숫자 배열을 오름차순으로 정렬하는 함수"
2. **구체적으로**: 입력/출력 타입, 예외 처리 포함
3. **컨텍스트 제공**: 기존 코드 스타일과 일관성 유지
